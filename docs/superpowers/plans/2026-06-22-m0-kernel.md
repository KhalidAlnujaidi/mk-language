# M0 — The Smallest Real Kernel · Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the smallest end-to-end kinox slice that is real: a hook can submit a prompt, the groom pipeline redacts/expands/adds-context/tags it (routing the one fuzzy step to the smallest fitting model), every boundary is recorded as an append-only `EventRecord`, and an immediate user correction is captured as a free label.

**Architecture:** Pure kernel (`manifest`, `router`, `metrics`, `corrections`) built only on the existing `kernel/contracts.py`. One product (`products/groom`) composes ground-truth stages plus the single fuzzy `tag` step through the router. One adapter (`adapters/claude_code.py`) is the only Claude-specific file. The `kx` entrypoint stubs become real for `doctor` (manifest health) and `new` (scaffold). The kernel imports nothing from `products/`/`adapters/`/`daemon/` — enforced by `tests/test_architecture.py`.

**Tech Stack:** Python 3.11+, stdlib only in the kernel (no new runtime deps — `pyproject.toml` `dependencies = []` stays empty), `uv` for env/test/lint/type, pytest + ruff + pyright (strict).

## Global Constraints

- **Kernel purity:** modules under `kernel/` import nothing from `products/`, `adapters/`, `daemon/`. `tests/test_architecture.py` must stay green.
- **No new runtime dependencies.** Kernel and product code use stdlib only (subprocess, json, os, pathlib, re, dataclasses). Dev-only deps stay as-is.
- **`null`, never a fabricated `false`/`0`.** A capability that cannot be measured is `None` (unknown), never `0` or `False` (verified-absent). Applies to every probe in `manifest.py`.
- **Honest observability.** `tokens_exact=False` for any cloud/estimated count; only exact local counts may set `tokens_exact=True`.
- **Fail-direction is explicit (thesis #2).** A guard (redact) fails CLOSED (block on doubt). An optimizer (expand, context, tag) fails SOFT (pass through on doubt). Each component states its `FailDirection`.
- **Ground truth beats the model (thesis #1).** `redact`, `expand`, `context` are deterministic code (no model). Only `tag` may call a model, and only the smallest fitting one.
- **`kin` is the reserved scope** — already rejected in `kx`; keep it rejected.
- **TDD, frozen dataclasses, full type annotations** (pyright strict). Use `from __future__ import annotations` at the top of every module, matching `kernel/contracts.py`.
- **Style:** match `kernel/contracts.py` — module docstring that cites the relevant vision §/thesis, `# --- Section ---` banners, factory classmethods over raw construction.

**Reference reading for every implementer:** `kernel/contracts.py` (the types you build on), `tests/test_contracts.py` (the house test style), `tests/test_architecture.py` (the guardrail), `alignment/CLAUDE.md` (conventions).

**Parallelism note for the controller:** Tasks 1–4 depend only on `kernel/contracts.py` and touch disjoint files — they are the parallel "swarm" wave (safe to run concurrently in isolated worktrees). Tasks 5–7 are the sequential integration wave (5 needs 1; 6 needs 1+4+5+2; 7 needs 6+2+3+1).

### Task 1: `kernel/manifest.py` — probe the machine, list available tiers

**Files:**
- Create: `kernel/manifest.py`
- Test: `tests/test_manifest.py`

**Interfaces:**
- Consumes: `kernel.contracts.Tier` (factories `Tier.deterministic()`, `Tier.model(name, where=...)`).
- Produces:
  - `@dataclass(frozen=True) class LocalModel: name: str; vram_gb_required: float | None`
  - `@dataclass(frozen=True) class Manifest:` fields `cpu_count: int | None`, `ram_gb: float | None`, `gpu_vram_gb: float | None`, `local_models: tuple[LocalModel, ...]`, `cloud_available: bool`
  - `Manifest.fitting_local_models() -> tuple[LocalModel, ...]` — models whose `vram_gb_required` is known **and** `<= gpu_vram_gb`. A model with `vram_gb_required is None` is NOT counted as fitting (cannot verify → not claimed). If `gpu_vram_gb is None`, no local model is claimed to fit (returns `()`).
  - `Manifest.available_tiers() -> tuple[Tier, ...]` — always includes `Tier.deterministic()` first; then one `Tier.model(m.name, where="local")` per fitting local model (smallest `vram_gb_required` first); then `Tier.model(<cloud_default>, where="cloud")` iff `cloud_available`. `<cloud_default>` = the constant `CLOUD_DEFAULT_MODEL = "claude-haiku-4-5"`.
  - `probe() -> Manifest` — runs all probes and assembles a `Manifest`. Pure-ish: shells out to read the machine; never raises on a missing tool (returns `None` for that field).
  - Helper probes, each returning `None` on any failure (missing binary, parse error, non-zero exit): `_probe_cpu() -> int | None`, `_probe_ram_gb() -> float | None`, `_probe_gpu_vram_gb() -> float | None`, `_probe_local_models() -> tuple[LocalModel, ...]`, `_probe_cloud() -> bool`.

**Probe details (stdlib only):**
- CPU: `os.cpu_count()`.
- RAM: parse `/proc/meminfo` `MemTotal:` (kB) → GB; `None` if the file is absent/unparseable.
- GPU VRAM: `subprocess.run(["nvidia-smi","--query-gpu=memory.total","--format=csv,noheader,nounits"], ...)`; parse first line MiB → GB; `None` if binary missing / non-zero exit / unparseable. Use `shutil.which("nvidia-smi")` first; `None` if absent.
- Local models: `ollama list` via `shutil.which("ollama")` then `subprocess.run(["ollama","list"], ...)`; parse model names from column 1 (skip header). `vram_gb_required` is unknown for now → `None` for each. Empty tuple if ollama missing.
- Cloud: `True` iff any of these env vars is set and non-empty: `ANTHROPIC_API_KEY`, `CLAUDE_API_KEY`. Else `False`. (Cloud presence is genuinely knowable, so `bool` is honest here.)

### Task 2: `kernel/metrics.py` — append-only EventRecord sink

**Files:**
- Create: `kernel/metrics.py`
- Test: `tests/test_metrics.py`

**Interfaces:**
- Consumes: `kernel.contracts.EventRecord`.
- Produces:
  - `class MetricsSink:` constructed `MetricsSink(path: Path)`.
  - `record(self, event: EventRecord) -> None` — append exactly one JSON object as one line (`\n`-terminated) to `path`, creating parent dirs and the file if absent. Never rewrites existing lines (append-only). Uses `dataclasses.asdict(event)` + `json.dumps`.

### Task 4: `products/groom/stages/` — the three ground-truth stages

**Files:**
- Create: `products/groom/__init__.py`, `products/groom/stages/__init__.py`
- Create: `products/groom/stages/redact.py`, `products/groom/stages/expand.py`, `products/groom/stages/context.py`
- Test: `tests/test_groom_stages.py`

**Interfaces:**
- Consumes: `kernel.contracts.FailDirection`.
- Produces (each stage is a pure-ish deterministic function; no model calls — thesis #1):
  - `redact.FAIL_DIRECTION = FailDirection.CLOSED` and `redact.redact(text: str) -> RedactResult` where `@dataclass(frozen=True) class RedactResult: text: str; found: tuple[str, ...]` — `text` has every detected secret replaced with `«REDACTED:{kind}»`; `found` lists the kinds detected (e.g. `"anthropic_key"`, `"openai_key"`, `"aws_key"`, `"generic_hex_token"`). Patterns (regex): Anthropic `sk-ant-[A-Za-z0-9_-]{20,}`, OpenAI `sk-[A-Za-z0-9]{20,}`, AWS access key `AKIA[0-9A-Z]{16}`, generic long hex `\b[0-9a-fA-F]{32,}\b`. Guard semantics: detection is deterministic; redaction always succeeds (replacement is total), so this stage never needs to block on its own — but it exposes `FAIL_DIRECTION = CLOSED` so the pipeline can fail closed if a future detector is uncertain.
  - `expand.expand(text: str) -> ExpandResult` where `@dataclass(frozen=True) class ExpandResult: text: str; notes: tuple[str, ...]` — finds `@`-mentions of filesystem paths (`@token` where token looks like a path), and appends a note per mention stating whether the path exists (`"@README.md → exists"` / `"@nope.txt → missing"`). Optimizer → `FAIL_DIRECTION = FailDirection.SOFT`; on any error returns the input unchanged with empty notes.
  - `context.gather(cwd: Path) -> ContextResult` where `@dataclass(frozen=True) class ContextResult: lines: tuple[str, ...]` — deterministic git/fs context: current branch (`git -C cwd rev-parse --abbrev-ref HEAD`) and the count of changed files (`git -C cwd status --porcelain` line count), each as a line like `"git.branch=m0-kernel"`, `"git.changed_files=3"`. Optimizer → `FAIL_DIRECTION = FailDirection.SOFT`; returns `ContextResult(())` if cwd is not a git repo or git is missing (never raises).

### Task 6: `products/groom/tag.py` + `products/groom/pipeline.py` — the squire pipeline

**Files:**
- Create: `products/groom/tag.py`, `products/groom/pipeline.py`
- Test: `tests/test_groom_pipeline.py`

**Interfaces:**
- Consumes: stages from Task 4, `kernel.router.route` (Task 5), `kernel.manifest.Manifest` (Task 1), `kernel.metrics.MetricsSink` (Task 2), `kernel.contracts.{Task, TaskKind, Annotation, EventRecord, Tier}`.
- Produces:
  - `tag.tag(text: str, manifest: Manifest) -> TagResult` where `@dataclass(frozen=True) class TagResult: tags: tuple[str, ...]; tier: Tier` — builds `Task(kind=TaskKind.TAG, budget_ms=TAG_BUDGET_MS)` (`TAG_BUDGET_MS = 200`), calls `route`. If the tier is a model tier, **for M0 do not actually call a model** — produce a deterministic keyword-based tag set (fail SOFT) and label the tier returned by the router so the boundary record is honest. If `route` returns `None`, use `Tier.deterministic()` and the same keyword fallback. Keyword tagging: lowercase the text, emit any of `{"bug","feature","refactor","test","docs","question"}` whose trigger words appear (e.g. "fix"/"error"→bug, "add"/"implement"→feature, "why"/"how"/"?"→question); empty tuple if none match. (The real model call lands post-M0; the router wiring and EventRecord must be real now.)
  - `pipeline.groom(prompt: str, *, manifest: Manifest, sink: MetricsSink, cwd: Path, task_id: str) -> Annotation` — runs redact → expand → context → tag in order; records one `EventRecord` per stage to `sink` (kind = stage name, tier = `"deterministic"` for the three ground-truth stages and the actual routed tier string for tag, `latency_ms` measured with `time.perf_counter`, `tokens_exact=False`). Returns `Annotation`: if redaction found secrets, still pass through but prepend a context line `"groomed: redacted {kinds}"` (redaction is total, so no block needed in M0); add expand notes, context lines, and `"tags: a, b"` to `Annotation.lines`. The only path that would `Annotation.halt(...)` in M0 is reserved/empty — keep `block=None`. Tier string format: `"deterministic"` or `f"model:{tier.where}"`.
