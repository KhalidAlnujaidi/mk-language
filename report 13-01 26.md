# Session Report — Making the enigma Dashboard Hub Publicly Accessible

**Date:** 2026-06-26 (Friday) · **Time:** 13:01
**Machine:** enigma (`concomp`, tailnet IP `100.64.164.41`)
**Goal:** Expose the internal dashboard hub (`http://100.64.164.41:8754/dashboard.html`) to the **public internet** without visitors needing to join the Tailscale network — and make it a permanent, 24/7 service.

---

## 1. Starting point

- The dashboard hub was a Python `http.server` (`dashhub.py`) bound to `0.0.0.0:8754`, reachable only on the LAN / tailnet.
- `100.64.164.41` is a Tailscale CGNAT (`100.64.0.0/10`) address → only reachable inside the tailnet.
- The hub lists/links many other dashboards via a live port scan + `registry.json`.
- It was running from a **volatile `/tmp` scratchpad** (would die on reboot/cleanup).

## 2. Approach chosen

Evaluated three options:

| Option | Ease | Notes |
|---|---|---|
| **Tailscale Funnel** ✅ chosen | One command, free, auto-HTTPS, no domain | Public URL is the `*.ts.net` MagicDNS name |
| Funnel-only (hub page) | Trivial | Deep links would break for off-tailnet visitors |
| Cloudflare Tunnel | More setup (domain + account) | Cleaner per-app subdomains; not needed here |

**Key problem found:** the hub built every link as `location.hostname:port` (e.g. `:8800`, `:6333`). Funnel only exposes **one** public host (ports 443/8443/10000), so those deep links would be dead for public visitors. Funnel alone = hub page public but all card links broken off-tailnet.

**Solution:** make the hub a **single public front door** that reverse-proxies the safe dashboards same-origin under `/d/<port>/`. One funnel, everything HTTPS, links work on both tailnet and public.

## 3. Code changes (`/home/enigma/dashhub/`)

- **`dashhub.py`**
  - Added a registry-allowlisted reverse proxy: `/d/<port>/<rest>` → `http://127.0.0.1:<port>/<rest>` (GET/HEAD/POST, redirect handling, `<base>` injection for self-contained pages). Allowlist prevents open-proxy/SSRF.
  - `scan()` now emits a `pubpath` per dashboard (same-origin public path, or `None` for tailnet-only).
  - Added `/setup` route serving a copy-paste commands page.
- **`registry.json`**
  - `"public": false` on **Qdrant** (SPA hard-codes `/dashboard/assets` + root API calls → can't live under a subpath) and **Sunshine** (login/stream).
  - **Ollama** stays `local_only` (never public — it's the GPU/LLM API).
  - Added static `Setup — make public` entry.
- **`dashboard.html`**
  - Links now use relative `pubpath` (works on tailnet AND funnel); cards badged **public** vs **tailnet-only**.
- **`setup.html`** (new)
  - Per-command copy boxes with a clipboard fallback (`navigator.clipboard` is blocked over plain http, so falls back to `execCommand`).

## 5. Tailscale Funnel setup

Blockers hit and resolved (the `!`/Claude channel cannot supply a sudo password — needs a real terminal):

1. **Operator set** (one-time, real SSH terminal): `sudo tailscale set --operator=$USER` → `OperatorUser: enigma`. No sudo needed for Tailscale after this.
2. **Funnel enabled on tailnet** (one-time, browser): `https://login.tailscale.com/f/funnel?node=...` → clicked **Enable**.
3. **Funnel started:** `tailscale funnel --bg 8754`.

**Result:**
```
https://concomp.tailfe113f.ts.net (Funnel on)
|-- / proxy http://127.0.0.1:8754
```

- Public cert is valid Let's Encrypt, chain: leaf → `YE1` → `ISRG Root YE` → `ISRG Root X2` → `ISRG Root X1` (browser-trusted).
- Note: inside-tailnet curl returns a self-signed/404 artifact; Node-based `WebFetch` rejects LE's brand-new ECDSA root. **Real browsers work** — definitive test is a phone on cellular.

## 6. Permanent 24/7 service

- App moved from the `/tmp` scratchpad to **`/home/enigma/dashhub/`** (`dashhub.py`, `dashboard.html`, `setup.html`, `registry.json`).
- Installed as a **user systemd service** (no sudo — linger already enabled):
  - Unit: `~/.config/systemd/user/dashhub.service` (`Restart=always`, `WantedBy=default.target`).
  - `systemctl --user enable --now dashhub.service` → `active (running)`, `enabled`.
  - `Linger=yes` → runs without login and across reboots.
  - Funnel config persists in tailscaled across reboots.
- **Verified self-healing:** killed the process → systemd respawned it in ~3s → back to HTTP 200.

## 7. Final state — live public links

```
https://concomp.tailfe113f.ts.net/          ← hub
https://concomp.tailfe113f.ts.net/setup     ← copy-paste commands
https://concomp.tailfe113f.ts.net/d/8800/   ← kinox Council
https://concomp.tailfe113f.ts.net/radar/    ← radar Brief
```

The funnel maps **one host → one local port**: `https://concomp.tailfe113f.ts.net` → `http://127.0.0.1:8754`. The hub fans out to other dashboards internally via `/d/<port>/`. Same URL works on the public internet and on the tailnet.

## 8. Managing it later (normal shell, no sudo)

```bash
systemctl --user status dashhub      # check
systemctl --user restart dashhub     # after editing files in /home/enigma/dashhub
tailscale funnel reset               # take public site OFFLINE
tailscale funnel --bg 8754           # back ONLINE
tailscale funnel status              # show mapping
```

**Add a new public dashboard:** add it to `registry.json` (public by default) and `systemctl --user restart dashhub`. Set `"public": false` to keep it tailnet-only.
